API Comparison

1. Convenience API - DataPoint[]

// Simple array of data points
const data: DataPoint[] = [
  { x: 0, y: 10 },
  { x: 1, y: 20 },
  { x: 2, y: 15 }
];

// Easy to use, familiar API
chart.appendData(0, data);

2. Performance API - Float32Array (Zero-Copy)

import { packDataPoints } from 'chartgpu';

const data: DataPoint[] = [
  { x: 0, y: 10 },
  { x: 1, y: 20 },
  { x: 2, y: 15 }
];

// Pack to Float32Array [x0,y0,x1,y1,...]
const packed = packDataPoints(data);

// Zero-copy transfer (best performance)
chart.appendData(0, packed, 'xy');
When to use Convenience API:
• Small datasets (<1K points per update)
• Infrequent updates (e.g., user interactions)
• Prototyping and development
• Code readability is priority

When to use Performance API:
• Large datasets (>10K points per update)
• High-frequency streaming (e.g., real-time sensors)
• Production applications with strict performance requirements
• Worker thread communication (eliminates serialization overhead)
